Documentación técnica para el despliegue de imágenes y encendido remoto en infraestructura educativa (Contexto: Red Andared Corporativo).

Objetivo: Permitir que el servidor Proxmox (ubicado en VLAN de Servidores) encienda los equipos del aula (VLAN Alumnos) saltando las restricciones de Broadcast de la red corporativa.

2. Requisitos Previos

  • Servidor Proxmox: Con acceso a la red del centro.
  • Raspberry Pi: Modelo 3, 4 o Zero (con adaptador Ethernet). Conectada por cable a la red del aula.
  • Red: Cableada (Ethernet). Evitar Wi-Fi para WoL.
  • Listado de MACs: Direcciones físicas de las tarjetas de red de los alumnos.

3. Configuración del Relay (Raspberry Pi)

La Raspberry actuará como el "agente infiltrado" en el aula.

1 Instalación de herramientas: Accede a la terminal de la RPi e instala el paquete necesario.

sudo apt update
sudo apt install wakeonlan

2 Fijar IP (Opcional pero recomendado): Asegura que la RPi siempre tenga la misma IP para que Proxmox la encuentre.

# Editar /etc/dhcpcd.conf
interface eth0
static ip_address=192.168.X.250/24
static routers=192.168.X.1
static domain_name_servers=192.168.X.1

4. Configuración del Servidor (Proxmox)

Configuramos la confianza SSH para no requerir contraseñas.

En la consola de Proxmox:

# 1. Generar claves (si no existen)
ssh-keygen -t rsa

# 2. Copiar clave a la Raspberry Pi
# Sustituir 'pi' por el usuario de la RPi y la IP correspondiente
ssh-copy-id pi@192.168.X.250
Prueba: Ejecuta ssh pi@192.168.X.250 desde Proxmox. Debería entrar sin pedir contraseña.

5. Script de Automatización

Este script reside en Proxmox. Lee un archivo de texto con las MACs y envía la orden a la Raspberry.

5.1 Archivo de MACs (lista_aula1.txt)

00:11:22:33:44:55
AA:BB:CC:DD:EE:FF
12:34:56:78:90:AB

5.2 Script Maestro (wol_aula.sh)

#!/bin/bash

# --- CONFIGURACIÓN ---
RPI_USER="pi"
RPI_IP="192.168.X.250"     # IP de la Raspberry en el aula
MAC_FILE="lista_aula1.txt" # Archivo con las MACs
BROADCAST="192.168.X.255"  # Dirección de Broadcast del aula
# ---------------------

echo "[INFO] Conectando con Agente WoL en $RPI_IP..."

while IFS= read -r mac; do
    # Ignorar líneas vacías
    if [[ ! -z "$mac" ]]; then
        echo " > Despertando: $mac"
        # Ejecución remota del comando wakeonlan forzando interfaz
        ssh $RPI_USER@$RPI_IP "wakeonlan -i $BROADCAST $mac"
    fi
done < "$MAC_FILE"

echo "[FIN] Secuencia completada."

Recuerda dar permisos de ejecución: chmod +x wol_aula.sh

6. Configuración de Clientes (BIOS/UEFI)

Para que funcione, los PCs de los alumnos deben estar configurados correctamente.

  • Power Management: "Resume by Alarm" o "Wake Up on LAN" -> ENABLED.
  • Deep Sleep / Low Power Mode: Debe estar DISABLED.
  • Boot Order (Para despliegue): Habilitar PXE / Network Boot como primera opción (o después del USB).

7. Solución de Problemas

Síntoma Causa Probable Solución
El script falla al conectar SSH Claves no copiadas o IP incorrecta Verificar ssh-copy-id y que la RPi tenga IP estática.
El comando se envía pero el PC no arranca Broadcast incorrecto o BIOS mal configurada Forzar la IP de broadcast en el comando (-i 192.168.X.255) y revisar "Deep Sleep" en BIOS.
Solo arrancan algunos equipos Cableado defectuoso Verificar que la luz del puerto Ethernet del PC parpadea (naranja/verde) incluso apagado.
