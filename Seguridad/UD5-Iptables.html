Iptables: Herramienta para Controlar el Tráfico de un Servidor [cite: 1]

Esta herramienta configurable tiene dos funciones básicas: [cite: 2]

  • Bloquear el acceso no permitido desde redes externas [cite: 3]
  • Restringir las conexiones de la red local con el exterior. [cite: 3]

¿Qué es Iptables? [cite: 4]

Iptables es un módulo del núcleo de Linux que se encarga de filtrar los paquetes de red, determinando qué paquetes de datos llegan hasta el servidor y cuáles no. [cite: 5]

Iptables funciona a través de reglas. El usuario indica mediante instrucciones el tipo de paquetes permitidos, los puertos de recepción, el protocolo utilizado y cualquier información relacionada con el intercambio de datos entre redes. [cite: 6]

Cuando el sistema recibe o envía un paquete, se recorren las reglas en orden hasta encontrar una que cumpla las condiciones. [cite: 7] Una vez localizada, esa regla se activa realizando la acción indicada sobre el paquete. [cite: 8]

Tablas y Cadenas en Iptables [cite: 9]

En iptables existen tres grandes grupos de reglas o tablas, dentro de las que se definen cadenas predefinidas. Cualquier paquete pasará por una de ellas para determinar su tratamiento. [cite: 10, 11]

Tabla Filter: Reglas de Filtrado [cite: 12]

La tabla filter es donde se indican las reglas de filtrado fundamentales. [cite: 13] Esta tabla contiene tres cadenas predefinidas que controlan el flujo de paquetes en diferentes direcciones. [cite: 14]

INPUT Hace referencia a los paquetes que llegan al sistema desde el exterior [cite: 15, 20]
OUTPUT Hace referencia a los paquetes generados en el sistema para ser enviados al exterior. [cite: 16]
FORWARD Se aplica a los paquetes que pasan por el servidor pero no tienen como destino el propio servidor (actuando como router). [cite: 18]
